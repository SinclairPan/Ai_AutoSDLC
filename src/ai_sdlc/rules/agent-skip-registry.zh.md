# 代理执行偏差登记协议

当自动化代理或协作者发现未遵守 `pipeline.md`、宪章、任务范围或质量门禁时，必须在同一迭代内登记，并把需要框架修正的问题同步写入 `docs/framework-defect-backlog.zh-CN.md`。

## 登记原则

1. 记录可复现的事实、影响范围和验证证据，不记录主观评价。
2. 每条记录必须绑定 `wi_id`；没有工作项时先建立正式工作项。
3. 框架建议应优先落在可机器验证的 rule、workflow、tool 或 eval。
4. 修复完成后更新状态，并补充回归测试或真实命令证据。
5. 仅文档 / 仅需求沉淀任务必须先更新 `spec.md`、`plan.md`、`tasks.md`；除非正式任务已授权实现，不得修改 `src/`、`tests/`。
