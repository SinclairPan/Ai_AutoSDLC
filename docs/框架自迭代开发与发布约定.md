# AI-SDLC 框架自迭代开发与发布约定

## 开发入口

AI-SDLC 自身开发必须先读取项目根目录 `AGENTS.md`，并遵循：

- `src/ai_sdlc/rules/pipeline.md`
- `src/ai_sdlc/rules/git-branch-rule.md`
- `docs/product-contract.md`

默认代理为 Codex，项目 Shell 为 PowerShell。命令应优先使用项目虚拟环境：

```powershell
uv sync
uv run ai-sdlc adapter status
```

## 变更流程

1. 从当前项目事实确认需求和影响面；
2. 为新行为或缺陷编写失败测试；
3. 实现最小、可验证的变更；
4. 运行定向测试和静态检查；
5. 执行 `ai-sdlc run --dry-run` 与约束验证；
6. 运行本地对抗 PR 审查；
7. 完成全量测试、构建和制品 smoke；
8. 更新用户可见文档和交付证据。

## 版本真值

1.0.0 的版本面必须同时保持一致：

- `pyproject.toml`
- `src/ai_sdlc/__init__.py`
- `ai_sdlc/__init__.py`
- `uv.lock`
- `.github/workflows/release-build.yml`
- `.github/workflows/release-artifact-smoke.yml`
- `.github/workflows/windows-user-guide-e2e.yml`
- `src/ai_sdlc/core/verify_constraints.py`
- `README.md`
- `USER_GUIDE.zh-CN.md`
- `packaging/offline/README.md`
- `docs/pull-request-checklist.zh.md`

## 发布前验证

```powershell
uv run pytest -q
uv run ruff check src tests scripts
uv run ai-sdlc verify constraints
uv run python scripts/validate_public_release_identity.py .
uv build
```

离线交付还必须完成：

- `ai-sdlc-offline-1.0.0-windows-amd64.zip`；
- `ai-sdlc-offline-1.0.0-macos-arm64.tar.gz`；
- `ai-sdlc-offline-1.0.0-linux-amd64.tar.gz`；
- manifest 与 SHA256 校验；
- 安装脚本 smoke；
- `ai-sdlc --version` 与 `--help`；
- Codex + PowerShell 初始化；
- adapter status 与 dry-run；
- Windows、macOS、Linux 目标平台验证。

## 发布约束

- Git 仓库固定为 `https://github.com/SinclairPan/Ai_AutoSDLC`；
- 产物版本固定从 `pyproject.toml` 读取；
- 工作流上传前必须先完成平台内安装 smoke；
- 发布动作必须由有权限的维护者明确执行；
- 失败的门禁、构建或 smoke 不得被标记为通过；
- 凭据只能来自 CI Secret 或环境变量。

## 交付完成定义

代码、测试、规则、文档、版本、离线包和工作流必须描述同一套当前能力；所有阻断项均已关闭，且结果可从全新环境复现。
